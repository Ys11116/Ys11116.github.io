学习安排及队伍分析

发表于 2020-04-02

写在前面

今年由于疫情的原因，电赛推迟到了10月份左右。疫情闲在家里，学了一堆跟电赛无关的东西。距离电赛只有大约5个月的时间了。接下来的5个月除了要完成学校的学习之外最重要的事情就是专心准备电赛。

个人分析

作为一个大一的小白，上学期无论是参加电协杯还是平时的学习主要都集中在了功率方面，这次由于组队的原因，为了发挥队伍的最大优势，选了测控方向。个人掌握的相关技能也只有51单片机的基本使用和模电的基础知识。目前正在学习32单片机的相关知识，后面还需要抓紧学习其他测控方面的知识，以保证不拖队伍后腿，希望也能在电赛中取得不错的成绩。

队伍分析

队伍里除了我之外的另外两个人都是助攻测控方面的，已经掌握了32单片机的基本使用，对于其他模块的使用方面还有所欠缺。大致的学习策略是他们两个一个主攻32单片机，一个主攻常用模块的使用，我继续学习模电方面的相关知识，以作为硬件后盾。

困惑

由于自己太菜，目前对于测控的学习方向有些模糊。除了32单片机和一些模块的使用需要学习之外。其他方面一些零碎的知识到底需要学习什么有些不清楚。然后队伍的学习分配也不明白是否合理。

2、Client & Server （客户机和服务器）

  • Client：主要指获取并显示Web服务器内容的软件，当前最主要的软件时浏览器。
  • 常用浏览器：IE、Chrome、Firefox、Safari、Opera以及手机上的各类浏览器。
  • 使用时要注意浏览器的兼容性。
  • Server：提供Web服务的软件
  • IIS：windows系统自带，只能在windows下运行
  • Apache：开源软件，能够跨平台运行
  • Tomcat：支持JSP的Web服务器，具有跨平台性，也是开源软件，经常与Apache结合使用
  • Nginx：占用内存少，并发能力强，国内使用nginx的网站用户：百度、新浪、网易、腾讯等

3、TCP/IP & port

五层协议

  • 应用层：HTTP、POP3/IMAP、SSH、FTP
  • 传输层：TCP、UDP
  • 网络层：IP
  • 数据链路层：以太网协议、WIFI协议、PPP（拨号上网）
  • 物理层：以太网线缆、光纤、modems

IP（IPv4）

  • IP，Internet Protocol，网际协议
  • 计算机间传输数据的一个简单协议
  • 每个设备都有一个32位的IP地址，一般写成四组8位二进制数（0—255）

![image-20200408164309174](/Users/ys11116/Library/Application Support/typora-user-images/image-20200408164309174.png)

  • 查看IP地址：ipconfig（windows）、ifconfig（linux/Mac）

TCP

  • TCP，Transfer Control Protocol，传输控制协议
  • TCP是一种位于IP协议之上的面向连接的、可靠的、基于字节流的传输层通信协议
  • 多路复用：多个程序使用同一个IP地址，通过Port来实现

![image-20200408164741833](/Users/ys11116/Library/Application Support/typora-user-images/image-20200408164741833.png)

4、URL & DNS

DNS

  • DNS：Domain、Name、System，即域名系统
  • 因特网上作为域名和IP地址互相映射的一个分布式数据库，能够使用户更方便的访问互联网，而不用去记住能够被机器直接读取的IP数串

URL

  • URL：Uniform Resource Locator、
  • 即统一资源定位器，是可以从互联网上得到的资源的位置和访问方法的一种简洁的表示，是互联网上标准资源的地址。互联网上的每一个文件都有一个唯一的URL，它包含的信息指出文件的位置以及浏览器应该怎么处理它

![image-20200408170650492](/Users/ys11116/Library/Application Support/typora-user-images/image-20200408170650492.png)

  • Protocol主要有以下几种

![image-20200408170811217](/Users/ys11116/Library/Application Support/typora-user-images/image-20200408170811217.png)

HTTP

  • HTTP：Hypertext Transfer Protocol，超文本传输协议
  • 服务器能够理解，从浏览器发出的一组命令
  • GET：从服务器请求文件或资源（下载）
  • POST：将表单信息提交到服务器
  • PUT：上传文件到服务器

计算机网络发展的7个阶段

发表于 2020-03-25

计算机网络发展的7个阶段

20世纪50年代：批处理时代

​ 批处理（Batch Processing）是指事先将用户程序和数据装入卡带或磁带，并由计算机按照一定的顺序读取，使用户所要执行的这些数据能够一并批量得到处理的方式。
​ 缺点：价格昂贵体积巨大，无法在一般的办公场所中使用。操作复杂，使用门槛高。有时程序处理时间长，不便个人使用。

20世界60年代：分时系统时代

​ TSS（Time Sharing System）分时系统是指多个终端与同一个计算机连接，允许多个用户同时使用同一台计算机的系统。
​ 优点：分时系统的产生实现了“一人一机”的目的，可用性得到极大改善。
​ 分时系统的特点：多路性、独占性、交互性、及时性
​ 正是从这一时期开始，网络（通信）与计算机之间的关系逐渐浮出水面，小型计算机也随即产生，办公场所与工程也逐渐引入计算机。

20世纪70年代：计算机间通信时代

​ 背景：计算机性能有了飞速发展，体积趋于小型化，同时价格急剧下降。
​ 计算机与计算机之间由通信线路连接
​ 显著提高了计算机的可用性，人们不再局限于仅使用一台计算机进行处理，而是逐渐使用多台计算机分布式处理，最终一并得到返回结果。
​ 在这一阶段计算机更侧重于满足使用者的需求、架构更灵活的系统，且操作比以往更加人性化。

20世纪80年代：计算机网络时代

​ 20世纪70年代，人们开始实验基于分组交换技术的计算机网络，到了80年代，一种能够互联多种计算机的网络随之诞生，它能够让各式各样的计算机相互连接。
​ 其中窗口系统的发明，更是拉近了人们与网络之间的距离，使用户更加体会到了网络的便捷之处

20世纪90年代：互联网普及时代

​ 到了此时，那些专注于信息处理的公司和大学已为每一位员工或者研究人员分配了一台计算机，实现了“一人一机”的环境。
​ 打响了“瘦身”与“多供应商”的口号，其目的在于连接不同厂商的计算机建立一个成本更低的网络环境
​ 连接异构型计算机的网络通信技术就是现在我们所看到的互联网技术

2000年：以互联网为中心的时代

​ 互联网的普及和发展着实对通信领域产生了巨大的影响。
​ 通过IP网络，人们不仅可以实现电话通信，电视播放，还能实现计算机之间的通信、建立互联网
​ 互联网的便捷性也给人们带来了负面问题，计算机病毒的侵害，信息泄漏，网络诈骗等利用互联网的犯罪行为日益增多，人们已不再满足于“单纯建立连接”，而是更为追求“安全建立连接”的目标。

2010年：TCP/IP

​ 如前面的介绍，互联网是由许多独立发展的网络通信技术融合而成。能够使他们之间不断融合并实现统一的正是TCP/IP技术
