2、Client & Server （客户机和服务器）

  • Client：主要指获取并显示Web服务器内容的软件，当前最主要的软件时浏览器。
  • 常用浏览器：IE、Chrome、Firefox、Safari、Opera以及手机上的各类浏览器。
  • 使用时要注意浏览器的兼容性。
  • Server：提供Web服务的软件
  • IIS：windows系统自带，只能在windows下运行
  • Apache：开源软件，能够跨平台运行
  • Tomcat：支持JSP的Web服务器，具有跨平台性，也是开源软件，经常与Apache结合使用
  • Nginx：占用内存少，并发能力强，国内使用nginx的网站用户：百度、新浪、网易、腾讯等

3、TCP/IP & port

五层协议

  • 应用层：HTTP、POP3/IMAP、SSH、FTP
  • 传输层：TCP、UDP
  • 网络层：IP
  • 数据链路层：以太网协议、WIFI协议、PPP（拨号上网）
  • 物理层：以太网线缆、光纤、modems

IP（IPv4）

  • IP，Internet Protocol，网际协议
  • 计算机间传输数据的一个简单协议
  • 每个设备都有一个32位的IP地址，一般写成四组8位二进制数（0—255）

![image-20200408164309174](/Users/ys11116/Library/Application Support/typora-user-images/image-20200408164309174.png)

  • 查看IP地址：ipconfig（windows）、ifconfig（linux/Mac）

TCP

  • TCP，Transfer Control Protocol，传输控制协议
  • TCP是一种位于IP协议之上的面向连接的、可靠的、基于字节流的传输层通信协议
  • 多路复用：多个程序使用同一个IP地址，通过Port来实现

![image-20200408164741833](/Users/ys11116/Library/Application Support/typora-user-images/image-20200408164741833.png)

4、URL & DNS

DNS

  • DNS：Domain、Name、System，即域名系统
  • 因特网上作为域名和IP地址互相映射的一个分布式数据库，能够使用户更方便的访问互联网，而不用去记住能够被机器直接读取的IP数串

URL

  • URL：Uniform Resource Locator、
  • 即统一资源定位器，是可以从互联网上得到的资源的位置和访问方法的一种简洁的表示，是互联网上标准资源的地址。互联网上的每一个文件都有一个唯一的URL，它包含的信息指出文件的位置以及浏览器应该怎么处理它

![image-20200408170650492](/Users/ys11116/Library/Application Support/typora-user-images/image-20200408170650492.png)

  • Protocol主要有以下几种

![image-20200408170811217](/Users/ys11116/Library/Application Support/typora-user-images/image-20200408170811217.png)

HTTP

  • HTTP：Hypertext Transfer Protocol，超文本传输协议
  • 服务器能够理解，从浏览器发出的一组命令
  • GET：从服务器请求文件或资源（下载）
  • POST：将表单信息提交到服务器
  • PUT：上传文件到服务器
